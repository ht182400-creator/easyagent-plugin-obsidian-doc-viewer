# Doc Project — 架构设计

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                         Doc Viewer                           │
├─────────────────────────────────────────────────────────────┤
│  UI Layer                                                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │FileTree  │  │GraphView │  │Markdown  │  │Search    │    │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘    │
│       └─────────────┴─────────────┴─────────────┘            │
│                         │                                    │
│  State Layer (Zustand)  │                                    │
│  ┌──────────────────────┴──────────────────────┐             │
│  │ docStore: docs / graphData / selectedDocId   │             │
│  └──────────────────────┬──────────────────────┘             │
│                         │                                    │
│  Service Layer          │                                    │
│  ┌──────────┐  ┌────────┴─────┐  ┌──────────────┐          │
│  │fsScanner │  │graphBuilder  │  │searchIndex   │          │
│  │wikiLink  │  │sampleData    │  │vectorIndex   │          │
│  └──────────┘  └───────────────┘  └──────────────┘          │
└─────────────────────────────────────────────────────────────┘
```

## 数据流

1. **初始化**: `docStore.loadSampleDocs()` 加载文档数据
2. **索引**: 调用 `createSearchIndex()` 构建 FlexSearch 索引
3. **图谱**: 调用 `buildGraphData()` 生成 D3 数据
4. **交互**: 用户点击节点 → `selectDoc(docId)` → MarkdownPreview 更新
5. **搜索**: 用户输入 → `search(query)` → 更新 searchResults

## 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 框架 | React 18 | 生态成熟，与 EasyAgent 一致 |
| 构建 | Vite | 快速 HMR |
| 样式 | Tailwind CSS | 开发效率高 |
| 图谱 | D3.js | 灵活可控 |
| 搜索 | FlexSearch | 高性能本地全文索引 |
| 状态 | Zustand | 轻量，与 EasyAgent 一致 |
| Markdown | react-markdown | 可扩展 remark 插件 |

## 扩展点

- `fsScanner.ts`: 替换为真实文件系统扫描
- `vectorIndex.ts`: 接入本地 embedding 模型
- `App.tsx`: 未来可通过 props 接收外部数据源
